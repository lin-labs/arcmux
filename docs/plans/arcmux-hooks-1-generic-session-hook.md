## Problem

`internal/hooks/hooks.go:103` writes a **new per-session** Claude hook script
`~/.claude/hooks/arcmux-<sessionID>.sh` on every `CreateSession` (sessionID is
`s-<ts>`, so files are `arcmux-s-*.sh`). **62 stray files exist on disk now.**
`Cleanup(sessionID)` only removes a script if the *same daemon process* still
holds it in its in-memory `scripts` map — so any crash/restart orphans them,
and they accumulate forever.

Worse: the user's `~/.claude/settings.json` references **none** of these
scripts, so they never fire — they are pure cruft. The thing that actually
matters downstream is the **per-session JSONL** at
`OutputPath(sessionID)` = `<HookOutputDir>/arcmux-hooks-<sessionID>.jsonl`,
which `daemon.go:507` hands to `watcher.Watch(id, hookPath)`.

## Goal

One **generic, idempotent** hook script that derives its per-session JSONL path
from environment (`ARCMUX_SESSION_ID` + `ARCMUX_HOOK_OUTPUT_DIR`), with that env
delivered to the spawned agent via a **`/tmp/arcmux` file handoff** that the
tmux loader loads *safely* before launching claude/codex. Plus coded+tested
cleanup of the 62 legacy scripts. Watcher/status/ready unchanged.

## (1) Generic hook script (replaces the per-session generator)

One fixed-name, fixed-content file: `~/.claude/hooks/arcmux-session-hook.sh`.

```sh
#!/bin/sh
# Auto-generated by arcmux — generic session-aware hook. Idempotent: content
# is fixed, so re-install is a no-op. Safe to register globally: it no-ops
# unless an arcmux-spawned session set the env below.
[ -n "$ARCMUX_SESSION_ID" ] || exit 0
[ -n "$ARCMUX_HOOK_OUTPUT_DIR" ] || exit 0
out="$ARCMUX_HOOK_OUTPUT_DIR/arcmux-hooks-$ARCMUX_SESSION_ID.jsonl"
mkdir -p "$ARCMUX_HOOK_OUTPUT_DIR" 2>/dev/null
EVENT_TYPE="${CLAUDE_HOOK_EVENT_TYPE:-unknown}"
TOOL_NAME="${CLAUDE_TOOL_NAME:-}"
printf '{"event":"%s","tool":"%s","ts":"%s","session_id":"%s"}\n' \
  "$EVENT_TYPE" "$TOOL_NAME" "$(date -u +%FT%TZ)" "$ARCMUX_SESSION_ID" \
  >> "$out"
```

- **No per-session script files** ever again — one file, one name.
- The JSONL filename is **byte-identical** to today's `OutputPath`
  (`arcmux-hooks-<id>.jsonl`), so the watcher is unaffected (see §5).
- `Install(sessionID, "claude", hookDir)` now ensures this one script exists
  (idempotent write) and still **returns `OutputPath(sessionID)`** so
  `daemon.go` keeps calling `watcher.Watch(id, thatPath)` unchanged.

## (2) Env contract

| Var | Meaning | Producer | Consumer |
|---|---|---|---|
| `ARCMUX_SESSION_ID` | the arcmux session id (`s-<ts>`) | session creator | generic hook |
| `ARCMUX_HOOK_OUTPUT_DIR` | `cfg.Hooks.HookOutputDir` (= installer `OutputDir`) | session creator | generic hook |

These must be in the **spawned agent's environment** so the hook (a child of
claude) inherits them. Delivered via the `/tmp/arcmux` handoff (§3).

## (3) `/tmp/arcmux` env handoff (Boyan-preferred mechanism)

Direct tmux session-scoped env (`NewSessionWithEnvPaneID`) has a known
sibling-pinning hazard (documented in `setupTmuxPane`). Instead:

- **Session creator writes** `/tmp/arcmux/<sessionID>.env` at `CreateSession`,
  containing strict `KEY=VALUE` lines (`ARCMUX_SESSION_ID`,
  `ARCMUX_HOOK_OUTPUT_DIR`, and any other caller env we choose to forward).
  This is an **env data file, not a hook script** — it does not violate
  "no new per-session script files."
- **The tmux loader loads it first.** `prof.StartCommand`
  (`claude --dangerously-skip-permissions --remote-control`) is the command
  sent via `SendKeys` (daemon.go:554) — the loader. We prefix it:

  ```
  eval "$(arcmux hook-env <sessionID>)" ; claude --dangerously-skip-permissions --remote-control
  ```

  `arcmux hook-env <id>` is a **new trusted subcommand** that reads the env
  file, validates it (§4), and prints **arcmux-quoted** `export KEY=VALUE`
  lines. We `eval` arcmux's *own* validated/quoted output — we **never
  `source` the raw writable file**. This is the key safety property.

## (4) Security guardrails (mandatory — /tmp is world-shared)

`/tmp/arcmux` and the env files are created and validated defensively:

- **Dir**: `/tmp/arcmux` created `0700`, must be a real dir (not a symlink)
  **owned by the current uid**; if it exists with wrong owner/mode/symlink,
  `hook-env` refuses (returns error, prints nothing → loader proceeds with no
  injected env, hook safely no-ops).
- **File**: `<id>.env` written `0600` via `O_CREATE|O_EXCL|O_WRONLY` (or
  truncate-after-lstat); on read, `Lstat` must show a regular file, current-uid
  owner, mode `0600` (no group/world bits), not a symlink.
- **Parsing** (`hook-env`): each line must match `^[A-Z_][A-Z0-9_]*=`; key must
  be in an **allowlist** (`ARCMUX_*`); value must contain no NUL/newline.
  Output is emitted as `export KEY='<single-quote-escaped>'` (every `'` →
  `'\''`) so even a hostile value cannot break out of the quotes when
  `eval`-ed. **No code from the file is ever executed** — only key=value data
  is read and re-quoted by arcmux.
- Net: an attacker who wins a race to write `/tmp/arcmux/<id>.env` can at worst
  set an allowlisted `ARCMUX_*` var to a string value (which only changes where
  the hook writes its own JSONL) — they **cannot inject shell**. Ownership/mode
  checks reject the file outright in the common case.

## (5) Watcher / status / ready compatibility

Unchanged by construction. `Install` keeps returning
`OutputPath(sessionID)` = `<OutputDir>/arcmux-hooks-<sessionID>.jsonl`;
`daemon.go` keeps `watcher.Watch(id, hookPath)`; the generic hook writes to that
exact path because `ARCMUX_HOOK_OUTPUT_DIR == OutputDir` and the filename format
is identical. `LatestEvents`/status/ready read the same JSONL. A regression test
asserts `Install`'s returned path is still what the watcher tails.

## (6) Cleanup / migration of the 62 legacy scripts

A **coded, tested** `Installer.CleanupLegacyScripts(hookDir) (int, error)`:

- Globs `<hookDir>/hooks/arcmux-s-*.sh` (the documented stray pattern) and
  removes each; explicitly **never** removes the generic
  `arcmux-session-hook.sh` (different prefix, plus an explicit skip guard).
- Idempotent: no error when zero match; returns the count removed.
- Called once on daemon startup (and/or first `Install`), so the 62 existing
  files are swept without a manual `rm`. Logged at info with the count.
- **Operational note for the PR/merge:** on first run of the new daemon, the 62
  `~/.claude/hooks/arcmux-s-*.sh` are deleted automatically by this code path;
  no operator action required. (A one-liner `rm` is *not* used; the sweep is in
  arcmux so it self-heals on every machine.)

## Test plan (Go, `go test ./...`)

1. **Generic install idempotency** — `Install` twice for two different sessions
   writes exactly one `arcmux-session-hook.sh` with fixed content; second call
   is a no-op (same bytes); no `arcmux-s-*.sh` created.
2. **Env-derived path** — run the generic hook (`sh arcmux-session-hook.sh`)
   with `ARCMUX_SESSION_ID`/`ARCMUX_HOOK_OUTPUT_DIR` set + `CLAUDE_*` set; assert
   it appends a valid JSON line to `<dir>/arcmux-hooks-<id>.jsonl`; and that with
   env unset it no-ops (writes nothing, exit 0).
3. **tmux-loader env loading** — `hook-env` against a well-formed
   `/tmp/arcmux/<id>.env` prints the expected quoted exports; `eval`-ing them in
   a subshell yields the right values; a **malicious** value
   (`X='; rm -rf /; '`) round-trips as a literal string (no execution).
4. **Security rejects** — `hook-env` refuses a world/group-writable file, a
   symlink, a wrong-owner file, a non-allowlisted key, and a value with a
   newline; prints nothing and errors.
5. **Cleanup** — seed N `arcmux-s-*.sh` + the generic; `CleanupLegacyScripts`
   removes the N, keeps the generic, returns N; second call returns 0.
6. **Watcher compat** — `Install` return value still equals `OutputPath`, and a
   JSONL written by the generic hook at that path is tailed/surfaced by the
   watcher (or, at minimum, an assertion that the path the watcher is given is
   the path the hook writes).

  global registration is harmless — but wiring it is a separate operational
